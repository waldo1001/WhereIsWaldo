Waldo DESIGN SYSTEM v1.0

A calm, trustworthy visual system for a private family map.

One shared token vocabulary, rendered natively on iOS (SwiftUI) and Android (Material 3). Light and dark are both first-class. Every text pairing meets WCAG 2.1 AA — this is a safety utility used outdoors, in sunlight, sometimes in a hurry.

Map-first Colorblind-safe status ≥44pt touch targets System fonts only
Palette rationale

The brand hue is a deep teal (#00696E) — blue's trust and calm with a warmer, more human, less "corporate-security" cast than a pure navy. It reads as a caring utility, not a surveillance console.

A single indigo accent (#4C5FD5) handles secondary actions without ever competing with the functional colors. Red is reserved strictly for danger, green for arrivals/online, amber for "stale — attention, not error". One brand hue, one accent, three functional colors, warm neutrals. No rainbow.

07

Keeping shared tokens native

Android · Material 3

Map the semantic tokens onto an M3 ColorScheme (primary/onPrimary/surface/surfaceVariant/error). Elevation is real dp with tonal overlay; buttons ripple on press; shapes use the M3 large/medium/small families keyed to our radius scale. Type is Roboto via Typography. SectionHeader is a genuine M3 list subheader. Switches, sliders and the FAB-style "locate now" follow M3 anatomy — the tokens just supply the color.

iOS · HIG

Same tokens drive a SwiftUI theme, but depth comes from the shadow-spec form of each elevation level (no Material overlay). Use SF Symbols for status glyphs and nav chevrons, Dynamic Type from the SF Pro sizes, and HIG spacing (16pt gutters, grouped-inset lists for rows/settings). The bottom sheet on the map is a native detent sheet; toggles are UISwitch-style pills. Pressed states dim rather than ripple.
